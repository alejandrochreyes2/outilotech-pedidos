### Sesión 5 — Corrección video R2 y URL raíz

#### Problemas encontrados

---

#### Fix 1 — Video hero no cargaba

**Causa raíz:** El path en el HTML tenía `/hero-tech.mp4` pero el archivo en R2 fue subido en la ruta `videos/hero-tech.mp4`.

| | Antes (roto) | Después (correcto) |
|---|---|---|
| HTML `<source>` | `r2BaseUrl + '/hero-tech.mp4'` | `r2BaseUrl + '/videos/hero-tech.mp4'` |
| URL resultante | `.../hero-tech.mp4` → 404 | `.../videos/hero-tech.mp4` → 200 ✅ |

**Archivo modificado:** [home.component.html](frontend/src/app/components/home/home.component.html) línea 13

---

#### Fix 2 — URL mostraba `/home` en el explorador

**Causa raíz:** La ruta raíz `''` hacía `redirectTo: '/home'`, lo que cambiaba la URL visible del explorador a `/home`.

**Solución en** [app.routes.ts](frontend/src/app/app.routes.ts):

| Ruta | Antes | Después |
|------|-------|---------|
| `''` | `redirectTo: '/home'` | `component: HomeComponent` (sirve directo) |
| `'home'` | `component: HomeComponent` | `redirectTo: '/'` (compatibilidad links) |
| `'**'` | `redirectTo: '/home'` | `redirectTo: '/'` |
| Otras (mac, iphone…) | `redirectTo: 'home'` | `redirectTo: '/'` |

**Resultado:**
- `outiltech.co/` → URL queda como `outiltech.co/` ✅
- `outiltech.co/home` → redirige a `outiltech.co/` ✅
- Todos los `routerLink="/home"` existentes siguen funcionando (el router hace el redirect) ✅

### Sesión 6 — Unificación de datos: Supabase como fuente única de verdad

#### Objetivo
El dashboard en `outiltech.co/dashboard` mostraba solo 3 registros (los del PostgreSQL de Hetzner), mientras que AppSheet mostraba 8+ registros (los de Supabase). El administrador necesita que ambas fuentes muestren los mismos datos visualmente desde el frontend.

#### Arquitectura de datos (antes vs después)

| Flujo | Antes | Después |
|-------|-------|---------|
| Dashboard lee de | PostgreSQL Hetzner via `.NET API` | **Supabase REST API** directamente |
| Pedidos view lee de | PostgreSQL Hetzner via `.NET API` | **Supabase REST API** directamente |
| AppSheet lee de | Supabase | Supabase (sin cambio) |
| Checkout escribe en | PostgreSQL Hetzner → sync a Supabase (con bug de ID) | PostgreSQL Hetzner → sync a Supabase (bug corregido) |

#### Bug corregido: IDs conflictivos en sync Supabase

**Causa:** `SyncPedidoToSupabase()` enviaba el `id` del PostgreSQL local (`p.Id`) junto con `"Prefer": "resolution=merge-duplicates"`. Como ambas secuencias empiezan en 1, 2, 3…, los registros de Hetzner sobreescribían los de Supabase con el mismo número de ID.

**Fix en** [backend/pedidos/Program.cs](backend/pedidos/Program.cs):

```csharp
// ANTES (roto — enviaba id que colisionaba):
var payload = new {
    id           = p.Id,   // ← REMOVIDO
    cliente      = p.Cliente,
    // ...con "Prefer": "resolution=merge-duplicates"
};

// DESPUÉS (correcto — Supabase asigna su propio SERIAL):
var payload = new {
    // id eliminado — Supabase auto-asigna
    cliente      = p.Cliente,
    total        = p.Total,
    // ...con "Prefer": "return=minimal"
};
```
